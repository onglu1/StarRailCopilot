# 深渊三模式（虚构叙事/忘却之庭/末日幻影）自动化任务设计

> 本文档最初为虚构叙事设计，后扩展为三模式共用（2026-06-12 下半夜补充 MoC/AS，见文末「三模式扩展」）。

> 状态：已实现并端到端验证（2026-06-12 凌晨）。用户睡眠中委托全权开发，本文档记录设计决策、实测事实与假设。
> 勘察截图存档：`log/pf_dev/`（gitignore 内，资产源图从这里裁切）。
> 已知后续事项见文末「遗留与后续」。

## 需求（用户原话拆解）

1. 自动点掉提示（弹窗/遮罩只出现一次，点掉就没了）
2. 自动选择最高能挑战的关卡
3. 自动选择配队（"配对"理解为配队 + 每节点的增益效果，两者都做）
4. 进入战斗，跟 boss 开大（开自动战斗后游戏 AI 自动放大招，实测确认）

## 实测确认的游戏流程（2026-06-12 凌晨，本季「造象立说」，4 关，0/12 星起步）

```
page_guide(星际和平指南)
  → 逐光捡金 tab → 左侧导航列表选「虚构叙事」(OCR) → 右панель「传送」
  → [一次性] 赛季开场动画（无需点击，自动进入）
  → 虚构叙事关卡星图（左上 ◆虚构叙事 徽记；节点 01-04 散布，状态文字：
      尚未挑战 / 未解锁 / (通关后:星数)；右上「规则说明」；右下 ★n/12）
  → 点关卡节点 → 关卡准备屏（左上「虚构叙事/编队」，右下「进入故事」）
      每节点(=半场)需要：≥1 角色 + 1 增益，否则 toast
      「请每队至少选择1名角色和1项增益效果」
      - 增益：行尾「+」槽 → 增益三选一屏（荒腔走板，右下「装配效果」）
      - 配队：点行内圆形空槽 → 左侧选人面板（「角色列表」/「预设编队」tab）
        → 预设编队 tab → 点预设块 → 整队填入当前节点
  → 进入故事 → 加载屏 → [每次?] 机制说明遮罩（"点击空白处关闭"）
  → 小地图行走段（与老忘却之庭同款）：摇杆前进 + 普攻(A)触敌
      左侧显示 当前积分 + 获得30000/60000/80000分（=1/2/3星线）
  → 波次战斗（开 AUTO + 2x，大招自动施放）
  → [待确认] 半场切换方式 / 结算屏样式
  → 退出回关卡星图 → 重扫关卡（通关解锁下一关）
```

关键实测事实：

- **3 星通关高关卡可直接获得全部前置关卡奖励**（准备屏原文），所以"只打最高关"就是最优策略，与用户要求一致。
- 节点 1/2 不能复用同一角色；已上阵角色在预设列表中变灰。
- 用户账号有预设编队：日常 / 队伍2 / 队伍3（前两个角色不重叠，可直接用于两个节点）。
- 进入"虚构叙事"导航项无需新资产：`DUNGEON_NAV_LIST.select_row(KEYWORDS_DUNGEON_NAV.Pure_Fiction)` 是 OCR 驱动，上游关键词已有 `Pure_Fiction`。
- 上游已有 `handle_forgotten_hall_buff()` 含 `BUFF_Pure_Fiction` 资产（进场增益弹窗，路径与本设计的"准备屏装配增益"不同，作为兜底保留调用）。
- 上游 `tasks/forgotten_hall/` 的 tab 导航代码已烂（`dungeon_tab` switch 里没有 Treasures_Lightward 状态），不能直接复用，需自配 tab 资产。
- 地图行走用上游 `MapControlJoystick`：JOYSTICK 中心 (248,560)，A_BUTTON (1023,521,1083,581)，RUN_BUTTON 正常。
- 战斗模块复用上游：`is_combat_executing`/`handle_combat_state`(AUTO+2x)/弹窗 handler。

## 方案选择

**配队方案**（三选一）：

- A. 预设编队 tab 点预设块（选定 ✓）：质量高（用户自己配的队）、点击少、坐标稳定。
  配置项 Team1Preset/Team2Preset（默认 1/2，范围 1-3，避免滚动）。
- B. 角色列表取前 4/前 8（按等级排序）：无需用户预设，但队伍质量差、禁用态检测麻烦。作为后续可选 fallback，v1 不做。
- C. OCR 按角色名选人（仿 ForgottenHall Weekly 的 CharacterList）：工作量大，v1 不做。

**增益方案**：每节点固定选第一个增益卡 + 装配（v1）。检测"已装配"不做——重复装配幂等无害，流程上每次都装一遍，简单可靠。

**选关方案**：全屏 OCR 数字节点（复用 `ForgottenHallStage` 关键词 01-15，digit OCR）+ 同节点下方状态词（未解锁/尚未挑战/已通关星数）。取"非未解锁"的最大编号。假设：本季 4 节点单屏显示无需拖动（实测如此）。

**调度方案**：注册进 Weekly 组（与 Rogue 并列）。成功后延到下周一 04:00（虚构叙事双周轮换，按周延迟最简单且安全；空跑一次代价小）。

## 任务终止条件

- 最高节点（当前可见最大编号）已通关（有星数）且无更高未解锁节点 → 任务完成。
- 同一节点连续 2 次挑战星数无增长 → 停止并告警（队伍强度不足，避免无限循环）。
- 行走段 90 秒未触发战斗 → 退出重进（一次），再失败则报错。

## 实现期实测修正（2026-06-12 凌晨）

1. **PF 战斗界面没有标准位置的暂停/AUTO/2x 按钮**（24 帧战斗截图全部验证 `COMBAT_PAUSE`/`COMBAT_AUTO` 模板 0 匹配），上游 `is_combat_executing`/`handle_combat_state` 在 PF 内不可用。改为：
   - 战斗检测：左上角波次旗标 `PF_WAVE_FLAG`（share 资产，稳定 HUD）。
   - AUTO 开启：**停滞看门狗**——战斗中画面 55 秒无变化（缩略帧 absdiff < 12，停滞实测 ≈6，活跃实测 70+）才盲点一次 AUTO 位置。auto 已开则画面持续变化，永不误点（无切换风险）。2x 不碰（沿用用户游戏内设置）。
2. **OCR 对关卡大数字检测不稳定**（同帧不同裁剪区随机漏 1 个数字），改为双区域多趟扫描取并集 + 去重配对。
3. **预设编队点击会被节点焦点动画吞掉**：加 `pf_team_filled()` 像素校验（4 槽位亮度 std：满 69-88 / 空 ~21）+ 最多 3 次重试 + 进关前强校验。
4. 半场切换确认：节点1 战斗结束 → 回到小地图（队伍2）→ 需再次行走触发下半场 → 结算。状态机循环天然覆盖。
5. 3 星通关高关后弹「快速通关」确认弹窗 + 「获得物品」弹窗（补发前置关卡奖励与星数，0/12 → 9/12 实测），新增 `QUICK_CLEAR_CONFIRM` 资产处理。
6. 关卡状态词表（OCR）：`未解锁`(锁) / `尚未挑战`(可挑战) / `快速通关`、`积分NNNN`(已通关)。
7. 进入故事的双重前置在准备屏上的表现：每节点需 ≥1 角色 + 1 增益，否则 toast「请每队至少选择1名角色和1项增益效果」。
8. 上游可复用确认：`TELEPORT` 资产 0.999 匹配 PF 传送按钮（search 区域覆盖）；`handle_forgotten_hall_buff` 的 `BUFF_Pure_Fiction` 在进场时实际触发过一次（保留调用）。
9. 安卓 BACK 键（keyevent 4）可逐级退出 PF 界面（备用恢复手段）。

## 假设清单（用户醒来后可推翻）

1. "配对"= 配队+增益（而非仅其一）。
2. 增益选第一个即可（不做三选一策略）。
3. 配队用游戏内预设 1/2；若预设不存在或角色冲突导致无法进入，任务报错并提示用户建预设（不静默乱配队）。
4. 只打最高可挑战关，不补低关星（游戏机制已保证高关 3 星拿全部奖励）。
5. 不自动领取 ★/12 宝箱奖励（v1 范围外，后续可加）。
6. 周一 04:00 重调度（双周赛季，空跑一次无害）。
7. dev 用 `config/dev_pf.json`（onglu 副本、GameClient=android、全任务 Enable=false），不动用户真实配置；正式使用时用户在自己配置里启用 PureFiction 任务。

## 三模式扩展（忘却之庭 MemoryOfChaos / 末日幻影 ApocalypticShadow）

应用户要求（醒来后补充）：挑战策略默认改为 `lowest_first`（从最低难度逐层打，适配所有练度、每轮多攒测试用例），`highest_only` 保留为可选项；三任务同一配置结构。

**架构**：副本内状态机与配队/装配逻辑抽到共享基类（同在 `tasks/pure_fiction/` 下，PF 离线回归确认无损）：

- `tasks/pure_fiction/abyss.py` `AbyssCombatLoop` —— 走图/战斗/停滞看门狗/结算退出，参数化 `SETTLE_BUTTON`、`abyss_home_check()`、`abyss_mid_settle_handler()` 钩子。
- `tasks/pure_fiction/abyss_prep.py` `AbyssPrep` —— 预设配队（校验+重试）、增益/公理装配、进入循环，参数化槽位/条带/进入键。满槽判定阈值统一 std>33（未聚焦行变暗实测 44，空槽最高 22）。

**实测确认的三模式异同**：

| | 虚构叙事 | 混沌回忆 | 末日幻影 |
|---|---|---|---|
| 选关形态 | 星图节点+状态词 | 横向水晶+金星 | 底部横条+状态词 |
| 进入键 | 进入故事 | 进入「混沌回忆」 | 前往挑战（准备屏） |
| 每节点增益 | 荒腔走板（必装） | 无 | 终焉公理（必装，同款装配屏） |
| 走图段 | 有 | 有 | 有 |
| 半场间形态 | 直接回地图 | 直接回地图 | **中场结算屏**（退出/重新挑战/前往节点二） |
| 结算按钮 | 返回虚构叙事 | 返回忘却之庭 | 返回末日幻影 |
| 波次旗标/查看详情/点击空白处关闭 | ✓ | ✓（共享资产全部复用） | ✓ |

- MoC 主屏复用上游 `FORGOTTEN_HALL_CHECK`（实测匹配新版水晶界面，page_forgotten_hall 白捡；注意它同样匹配 MoC 准备屏，moc_in_stage_screen 用 `非 PREP_CHECK` 排除）。
- MoC 选关无状态词：金星像素判已通关（金色 (241,189,110)±30/30/40，单星≈150px、阈值80、紫色辉光背景=0），未解锁不可见——行为式降级（进不去准备屏→跳过该层）。
- AS 进入流程多两层：传送→赛季总览（前往挑战）→关卡横条选关（前往挑战）→**首领信息两页**（下一步/前往编队，一次性）→准备屏。
- AS 已通关条目显示积分（如 `01 7525`），解析器把纯数字≥3位视作已通关。
- AS 中场结算：优先点亮色「前往节点二」；若节点1失败无该按钮→点「退出」兜底；循环增加"意外回到主屏"防御分支。
- 指南页签非粘性（常开在每日实训），三任务的 goto 都显式点 tab。
- **星启模式（AS 难度4）不支持**：通关难度3后解锁，需挑战 3 个节点（3 套不重复编队，boss 等级90），且解锁时弹一次性教学轮播（上游 `handle_tutorial` 可处理，已挂进循环）。当前实现：进准备屏后 OCR 检出「星启」→ BACK 退出 → 跳过该关收尾。完整支持记 backlog。
- **e2e 实战教训**：MoC 战败屏/胜利结算屏都是单按钮，上游 `POPUP_SINGLE` 能兜底点掉（胜利屏现已有专用 `MOC_SETTLE_BACK` 资产走设计路径，战败屏继续靠兜底）；AS 通关后条目可能 OCR 成带空格的 `01 7525`，解析需 strip；战斗停滞看门狗升级：连点 3 次 AUTO 无帧变化 → GameStuckError 走任务级重启（防误触模态后无限空点）。
- **手动驱动教训（不影响任务代码）**：战斗中盲发 BACK 会弹「确认退出游戏」；盲点击序列与游戏状态竞争会误触队友头像进入选择模态——任务代码每帧检查状态后才点击，无此风险。
